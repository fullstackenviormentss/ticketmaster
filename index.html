Ticketmaster provides a universal API to ticket tracking and project management systems. You can use it to connect an application to any ticketing system with a supported provider. You can also use the command line interface to easily manage your workflow when using one or more different back end project management or ticketing systems.

How To Install

Fire up terminal, then 

    gem install ticketmaster

then install whatever providers you want to use, for example: 

    gem install ticketmaster-pivotal

Examples

Here are a few simple examples to give you an idea of some of what you can do with Ticketmaster. The first one just gets a list of available projects for your account on Lighthouse. 

    require 'rubygems'
    require 'ticketmaster'
    require 'ticketmaster-lighthouse'

    # display list of projects
    tm = TicketMaster.new(:lighthouse)
    tm.projects.each {|project|
      puts "#{project.id} - #{project.name}"
    }

The next example gets a list of tickets for the first project for your account on Pivotal Tracker. 

    require 'rubygems'
    require 'ticketmaster'
    require 'ticketmaster-pivotal'

    # display list of tickets for last project
    tm = TicketMaster.new(:pivotal)
    project = tm.projects.last
    project.tickets.each {|ticket|
      puts "#{ticket.id} - #{ticket.title}"
    }

This third example copies all tickets and associated comments for a project from Lighthouse to Pivotal Tracker. 

    require 'rubygems'
    require 'ticketmaster'
    require 'ticketmaster-pivotal'
    require 'ticketmaster-lighthouse'

    # copy all tickets and comments from pivotal tracker to new lighthouse project
    pivotal = TicketMaster.new(:pivotal)
    pivotal_project = pivotal.project(97107)
    lighthouse = TicketMaster.new(:lighthouse)
    lighthouse_project = lighthouse.project!(:name => "Copy Test on #{Time.now}", 
                                             :description => "A copy test")

    pivotal_project.tickets.each do |pivotal_ticket|
      puts pivotal_ticket.title, pivotal_ticket.description

      lighthouse_ticket = lighthouse_project.ticket!({:title => pivotal_ticket.title, :description => pivotal_ticket.description})
      pivotal_ticket.comments.each do |comment|
        lighthouse_ticket.comment!(:text => comment.body)
      end
    end

Copying all tickets and associated comments, like example 3 above, is such a typical use case for Ticketmaster, that we have a shortcut: the copy method: 

    require 'rubygems'
    require 'ticketmaster'
    require 'ticketmaster-pivotal'
    require 'ticketmaster-lighthouse'

    # copy all tickets and comments from pivotal tracker to new lighthouse project (the easy way)
    pivotal = TicketMaster.new(:pivotal)
    pivotal_project = pivotal.project(97107)
    lighthouse = TicketMaster.new(:lighthouse)
    lighthouse_project = lighthouse.project!(:name => "Copy Test on #{Time.now}", 
                                             :description => "A copy test")
    
    lighthouse_project.copy(pivotal_project)

Command Line Interface

Ticketmaster includes a command line interface, to make it very easy to perform tasks like creating or closing tickets without disturbing your workflow. The tm command allows you to access all of the functionality of Ticketmaster. A configuration file, located at ~/.ticketmaster.yml by default, tells Ticketmaster about the various backend accounts you have setup. You can also use project-specific ticketmaster.yml files to connect to whichever back-end systems are in use for each project. Here is an example of getting a list of all projects for the default provider setup in ticketmaster.yml: 

    tm project --list

This example gets the list of all tickets for the default provider and default project as setup in ticketmaster.yml: 

    tm ticket --list

Supported Providers

Ticketmaster has a plugable architecture, that makes it pretty easy to write a new provider to support your preferred backend system. Here are the providers that we currently have support for:

* Pivotal Tracker (ticketmaster-pivotal)
* Lighthouse (ticketmaster-lighthouse)
* Basecamp (ticketmaster-basecamp)
* Github Issue Tracking (ticketmaster-github)
* Unfuddle (ticketmaster-unfuddle)
* Kanban Pad (ticketmaster-kanbanpad)
* Redmine (ticketmaster-redmine)

We are working on supporting lots more back-ends, and other people are too. Check back for the latest and greatest provider support here.
